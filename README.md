# ✈️ T-AGENCY — Портал управления бронированием туров

Веб-приложение на **Django 5.2** для подбора отелей и расчета стоимости туров. Основной акцент сделан на сложной бизнес-логике и продвинутой серверной архитектуре. 

## 🛠 Статус проекта: Полностью развернут и доступен для тестирования. 

🔗 **Живое демо:** [http://79.174.77.122/touragency_proj/](http://79.174.77.122/touragency_proj/)

---

## 🛠 Технологический стек
*   **Backend:** Django 5.2, Python 3.x, PostgreSQL.
*   **Frontend:** JavaScript (jQuery), AJAX, Bootstrap 5 (Responsive Design).
*   **Infrastructure:** Nginx, Gunicorn (Socket Activation), Systemd, Python-dotenv.

---

## 🚀 Ключевые технические особенности 
*   **Сложный расчет стоимости:** Реализована логика бронирования с учетом динамического числа гостей и фиксированных шагов в выборе срока проживания 
(бизнес-логика: 3, 7, 11 ночей).
*   **Универсальный AJAX-контроллер:** Единый интерфейс обновления параметров тура (ночи, гости, категория номера) с мгновенным пересчетом итоговой суммы через JsonResponse.
*   **Архитектура моделей:** Использование Custom QuerySets и динамического обновления атрибутов через setattr для чистоты кода и масштабируемости.
*   **UX Оптимизация:** Кастомные элементы управления инкрементом (+/-) с валидацией минимальных значений, адаптированные под мобильные устройства.
*   **Маски и валидация:** Клиентская валидация форм и маски ввода (телефон) на чистом JS.

## ⚙️ Deployment & Infrastructure (Advanced DevOps)
Проект развернут на **VPS (Ubuntu)** с архитектурой, готовой к продакшену:
*   **System Socket Activation:** В отличие от классического запуска, здесь реализована модель активации через сокет-юнит (.socket). Это обеспечивает создание сокета системой до старта приложения и бесшовную обработку очередей запросов. 
*   **Reverse Proxy (Nginx):** Маршрутизация через префикс /touragency_proj/ с использованием SCRIPT_NAME. Полная изоляция статических файлов (STATIC_ROOT) от исходных ресурсов.
*   **Security:** Конфигурация через `.env`, настройка `CSRF_TRUSTED_ORIGINS` для работы за прокси-сервером. Строгое разграничение прав доступа к UNIX-сокету (SocketGroup/SocketMode) 
*   **Static Automation:** Автоматизированная сборка артефактов статики в зависимости от режима DEBUG (Development/Production).

## 💻 Как запустить локально
1. Клонировать репозиторий: `git clone https://github.com/Archibald-prog/touragency_proj.git`
2. Создать виртуальное окружение: `python -m venv venv` 
    (для пользователей Mac OS - создать ВО с конкретной версией языка: `python3.12 -m venv venv`) 
3. Активировать ВО: `source venv/bin/activate` (в Windows PowerShell или cmd: `venv\Scripts\activate`) 
4. Установить зависимости: `pip install -r requirements.txt`
5. Создать базу данных и пользователя БД в своей локальной PostgreSQL (через клиент pgAdmin или терминал) 
6. Создать файл `.env` на основе `.env.example` (скопируйте шаблон настроек командой `cp .env.example .env`) 
7. Сгенерировать SECRET_KEY, отдав в терминале команду: 
    `python -c 'from django.core.management.utils import get_random_secret_key; print(get_random_secret_key())'` 
8. В файле `.env` указать свои локальные значения переменных - включая SECRET_KEY и параметры БД в переменной `DATABASE_URL`
9. Создать в корне проекта каталог `media` (отдать команду: `mkdir media`)
10. Применить миграции: `python manage.py migrate`
11. Заполнить базу тестовыми данными: `python manage.py loaddata fixtures/init_db.json`
12. Запустить сервер: `python manage.py runserver`
